My_Compiler 1.0版本说明：


（1）预处理器：本编译器直接对源代码进行编译，这样使得我们的精力更加关注于编译器的实现细节上
（2）采用一遍编译的方式
（3）删除了C语言中，对浮点数和struct类型，并将数组和指针的维数简化到一维。
（4）挑选若干编译优化算法作为优化器的实现

（5）汇编指令属于Intel X86处理器指令集的子集，虽然这间接降低了汇编器实现的复杂度，但不会影响汇编器关键流程的实现。
	 编译器在产生汇编代码之前已经分析了源程序的正确性，生成的汇编代码都是合法的汇编指令，因此在汇编器的实现过程中
	 不需要考虑汇编语言的词法、语法和语义错误的情况。
（6） 采用静态链接的方式
（7）ELF文件信息。除了ELF文件必须的段和数据，我们把代码全部存放在".text"段，数据存在".data"段

语义分析模块处理如下问题：
	（1）变量及函数使用前是否定义？
	（2）break语句是否出现在循环或switch-case语句内部？
	（3）continue语句是否出现在循环内部？
	（4）return语句返回值的类型是否与函数返回值类型兼容
	（5）函数调用时，实参列表和形参列表是否兼容？
	（6）表达式计算及赋值时，类型是否兼容？
